---
title: "Safety Checker"
description: "SafetyCheckerServer and SafetyCheckerClient — geofence validation and vehicle constraint enforcement over ZMQ."
---

The safety checker is a **ZMQ request-reply service** that validates vehicle commands against operator-configured geofences and flight constraints before they are executed. The server runs as a separate process; your script connects to it as a client.

```python
from aerpawlib.v1 import SafetyCheckerServer, SafetyCheckerClient
```

---

## SafetyCheckerServer

The server loads a YAML configuration file, reads KML geofence polygons, and listens for validation requests on a ZMQ REP socket.

<Warning>
The `SafetyCheckerServer` constructor **blocks indefinitely** — the server loop runs until the process is killed. Always run it in a separate process or thread.
</Warning>

### Constructor

```python
SafetyCheckerServer(vehicle_config_filename, server_port=14580)
```

<ParamField path="vehicle_config_filename" type="str" required>
  Path to the YAML configuration file. See [Configuration file format](#configuration-file-format) below.
</ParamField>

<ParamField path="server_port" type="int">
  TCP port to bind the ZMQ REP socket to. Defaults to `14580`.
</ParamField>

### Running as a subprocess (recommended)

```bash
python -m aerpawlib.v1.safety --port 14580 --vehicle_config geofence_config.yaml
```

### Running inline (blocks)

```python
from aerpawlib.v1 import SafetyCheckerServer

# This call never returns
SafetyCheckerServer("geofence_config.yaml", server_port=14580)
```

### Running from your script with ExternalProcess

```python
from aerpawlib.v1 import ExternalProcess, BasicRunner, entrypoint, Drone

class SafeMission(BasicRunner):
    @entrypoint
    async def run(self, drone: Drone):
        server = ExternalProcess(
            "python",
            params=["-m", "aerpawlib.v1.safety",
                    "--port", "14580",
                    "--vehicle_config", "geofence_config.yaml"],
        )
        await server.start()
        # ... rest of mission
```

## SafetyCheckerClient

Connects to a running `SafetyCheckerServer` over TCP/ZMQ and exposes validation methods. Supports use as a context manager.

### Constructor

```python
SafetyCheckerClient(addr, port, timeout_s=10.0)
```

<ParamField path="addr" type="str" required>
  IP address of the safety checker server (e.g. `"127.0.0.1"`).
</ParamField>

<ParamField path="port" type="int" required>
  Port the server is bound to (must match `server_port` on the server).
</ParamField>

<ParamField path="timeout_s" type="float">
  Timeout in seconds for send/receive operations. Prevents indefinite blocking if the server is down. Defaults to `10.0`.
</ParamField>

```python
client = SafetyCheckerClient("127.0.0.1", 14580)

# or as a context manager (calls close() automatically)
with SafetyCheckerClient("127.0.0.1", 14580) as client:
    ok, msg = client.check_server_status()
```

### `check_server_status`

```python
ok, msg = client.check_server_status()
```

Verify that the safety checker server is reachable and responding.

<ResponseField name="ok" type="bool">
  `True` if the server is up and healthy.
</ResponseField>

<ResponseField name="msg" type="str">
  Empty string on success; error description on failure.
</ResponseField>

```python
ok, msg = client.check_server_status()
if not ok:
    raise RuntimeError(f"Safety checker not running: {msg}")
```

### `validate_waypoint_command`

```python
ok, msg = client.validate_waypoint_command(current_location, next_location)
```

Verify that the straight-line path from `current_location` to `next_location`:
- stays within at least one include geofence,
- does not enter any exclude geofence,
- does not cross a geofence boundary, and
- (for copters) has an altitude within `[min_alt, max_alt]`.

<ParamField path="current_location" type="Coordinate" required>
  The vehicle's current position.
</ParamField>

<ParamField path="next_location" type="Coordinate" required>
  The intended target waypoint.
</ParamField>

<ResponseField name="ok" type="bool">
  `True` if the waypoint and path are valid.
</ResponseField>

<ResponseField name="msg" type="str">
  Empty string on success; human-readable reason on failure (e.g. `"Waypoint (35.72,-78.69) is outside of the geofence. ABORTING!"`).
</ResponseField>

```python
target = Coordinate(35.7275, -78.6960, 15)
ok, msg = client.validate_waypoint_command(drone.position, target)
if ok:
    await drone.goto_coordinates(target)
else:
    print(f"Waypoint rejected: {msg}")
    await drone.land()
```

### `validate_change_speed_command`

```python
ok, msg = client.validate_change_speed_command(new_speed)
```

Verify that `new_speed` is within the configured `[min_speed, max_speed]` range.

<ParamField path="new_speed" type="float" required>
  The requested groundspeed in m/s.
</ParamField>

<ResponseField name="ok" type="bool">
  `True` if the speed is within limits.
</ResponseField>

<ResponseField name="msg" type="str">
  Empty string on success; reason on failure (e.g. `"Invalid speed (25.0) greater than maximum (15)"`).
</ResponseField>

```python
ok, msg = client.validate_change_speed_command(12.0)
if ok:
    await drone.set_groundspeed(12.0)
else:
    print(f"Speed rejected: {msg}")
```

### `validate_takeoff_command`

```python
ok, msg = client.validate_takeoff_command(takeoff_alt, current_lat, current_lon)
```

Verify that the requested takeoff altitude is within the configured `[min_alt, max_alt]` range. Also records the takeoff location for later landing validation.

<ParamField path="takeoff_alt" type="float" required>
  Requested takeoff altitude AGL in meters.
</ParamField>

<ParamField path="current_lat" type="float" required>
  Current latitude in decimal degrees.
</ParamField>

<ParamField path="current_lon" type="float" required>
  Current longitude in decimal degrees.
</ParamField>

<ResponseField name="ok" type="bool">
  `True` if the altitude is within limits.
</ResponseField>

<ResponseField name="msg" type="str">
  Empty string on success; reason on failure.
</ResponseField>

```python
ok, msg = client.validate_takeoff_command(10, drone.position.lat, drone.position.lon)
if ok:
    await drone.takeoff(10)
else:
    print(f"Takeoff rejected: {msg}")
```

### `validate_landing_command`

```python
ok, msg = client.validate_landing_command(current_lat, current_lon)
```

Verify that the vehicle is within 5 meters of the recorded takeoff location (set by a prior call to `validate_takeoff_command`).

<ParamField path="current_lat" type="float" required>
  Current latitude.
</ParamField>

<ParamField path="current_lon" type="float" required>
  Current longitude.
</ParamField>

<ResponseField name="ok" type="bool">
  `True` if the landing position is acceptable.
</ResponseField>

<ResponseField name="msg" type="str">
  Empty string on success; reason on failure.
</ResponseField>

### `close`

```python
client.close()
```

Close the ZMQ socket and terminate the ZMQ context. Called automatically when used as a context manager.

## Configuration file format

The YAML configuration file passed to `SafetyCheckerServer` defines vehicle type, speed limits, altitude limits (copter only), and geofence files.

```yaml
# geofence_config.yaml
vehicle_type: copter        # "copter" or "rover"
max_speed: 15               # maximum allowed speed (m/s)
min_speed: 0.5              # minimum allowed speed (m/s)
max_alt: 120                # maximum altitude AGL (m) — copter only
min_alt: 5                  # minimum altitude AGL (m) — copter only
include_geofences:          # allowed flight areas (vehicle must stay inside)
  - flight_area.kml
exclude_geofences:          # no-go zones (vehicle must stay outside)
  - no_fly_zone.kml
  - building_zone.kml
```

### Required fields (all vehicle types)

| Field | Type | Description |
|---|---|---|
| `vehicle_type` | `str` | `"copter"` or `"rover"` |
| `max_speed` | `float` | Maximum allowed groundspeed in m/s |
| `min_speed` | `float` | Minimum allowed groundspeed in m/s |
| `include_geofences` | `list[str]` | Paths to KML files defining allowed flight areas |
| `exclude_geofences` | `list[str]` | Paths to KML files defining no-go zones |

### Additional fields for copters

| Field | Type | Description |
|---|---|---|
| `max_alt` | `float` | Maximum altitude AGL in meters |
| `min_alt` | `float` | Minimum altitude AGL in meters |

<Note>
KML file paths in the config are resolved **relative to the config file's directory**, not the working directory.
</Note>

## KML geofence format

Geofence files must be KML files containing a single `Placemark` with a `Polygon`. Coordinates are `lon,lat,alt` triples (the KML standard order).

```xml
<?xml version="1.0" encoding="UTF-8"?>
<kml xmlns="http://www.opengis.net/kml/2.2">
  <Document>
    <Placemark>
      <name>Flight Area</name>
      <Polygon>
        <outerBoundaryIs>
          <LinearRing>
            <coordinates>
              -78.700,35.720,0
              -78.690,35.720,0
              -78.690,35.730,0
              -78.700,35.730,0
              -78.700,35.720,0
            </coordinates>
          </LinearRing>
        </outerBoundaryIs>
      </Polygon>
    </Placemark>
  </Document>
</kml>
```

<Warning>
The polygon must be **closed** — the last coordinate must repeat the first. The parser expects exactly one `Placemark` per file.
</Warning>

## Full integration example

```python
from aerpawlib.v1 import (
    Drone, Coordinate,
    BasicRunner, entrypoint,
    SafetyCheckerClient,
    ExternalProcess,
)

class SafeFlight(BasicRunner):
    @entrypoint
    async def run(self, drone: Drone):
        # Launch safety checker server as a subprocess
        server = ExternalProcess(
            "python",
            params=["-m", "aerpawlib.v1.safety",
                    "--port", "14580",
                    "--vehicle_config", "config.yaml"],
        )
        await server.start()

        checker = SafetyCheckerClient("127.0.0.1", 14580)

        # Verify server is running
        ok, msg = checker.check_server_status()
        assert ok, f"Safety checker not running: {msg}"

        # Validate takeoff altitude
        pos = drone.position
        ok, msg = checker.validate_takeoff_command(15, pos.lat, pos.lon)
        assert ok, f"Takeoff rejected: {msg}"
        await drone.takeoff(15)

        # Validate each waypoint before navigating
        target = Coordinate(35.7275, -78.6960, 15)
        ok, msg = checker.validate_waypoint_command(drone.position, target)
        if ok:
            await drone.goto_coordinates(target)
        else:
            print(f"Waypoint outside geofence: {msg}")

        # Validate landing location
        ok, msg = checker.validate_landing_command(
            drone.position.lat, drone.position.lon
        )
        if ok:
            await drone.land()
        else:
            await drone.return_to_launch()

        checker.close()
```
